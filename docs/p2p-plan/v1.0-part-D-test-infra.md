## Your role

You ship the test infrastructure that makes Parts A/B/C verifiable as a
whole system, both locally and on Base Sepolia:

1. **Anvil bring-up helpers** — Go package that spawns/configures a local
   Anvil chain, deploys the v1.0 contracts on top, hands callers a
   ready-to-use RPC URL and `eth.Client`.
2. **Sepolia deploy scripts** — wrappers that drive Part A's
   `DeploySepolia.s.sol` from a CI-friendly entrypoint, manage testnet
   faucet ETH, record deployment addresses.
3. **Real-engine integration test** — the v1.0 analog of v0's
   `TestRealEngineIntegration`. Wires the real ledger, audit, and
   federation against on-chain backings on Anvil; runs a mixed scenario
   (90 honest + 10 malicious jobs); asserts on-chain state.
4. **CI integration** — GitHub Actions (or whatever the project uses)
   that runs Anvil tests on every PR.

You are the **only** orchestrator allowed to edit files under
`core/p2p/eth/testharness/` and `core/deploy/v1/`.

## 1. Anvil bring-up helpers

`anvil.go`:

```go
package testharness

type AnvilOpts struct {
    Port       int       // 0 → pick free port
    BlockTime  string    // "" → instant blocks
    PrintLogs  bool
    AccountSet AccountSet
}

type AccountSet struct {
    Founder       Account
    Workers       []Account
    Publishers    []Account
}

type Account struct {
    Address common.Address
    PrivateKey *ecdsa.PrivateKey
    BalanceETH float64
}

type Anvil struct {
    URL       string
    ChainID   *big.Int
    Cmd       *exec.Cmd
    Contracts eth.ContractAddresses // populated by Deploy()
    Accounts  AccountSet
}

func StartAnvil(t *testing.T, opts AnvilOpts) *Anvil  // calls t.Cleanup to tear down
func (a *Anvil) Deploy(t *testing.T) error             // shells out to forge script
func (a *Anvil) BootstrapGenesis(t *testing.T, cfg BootstrapConfig) error  // calls cmd/p2p-bootstrap or its lib equivalent
func (a *Anvil) Client() *eth.Client
```

Test usage:

```go
func TestSomething(t *testing.T) {
    anvil := testharness.StartAnvil(t, testharness.AnvilOpts{...})
    if err := anvil.Deploy(t); err != nil { t.Fatal(err) }
    if err := anvil.BootstrapGenesis(t, defaultGenesis()); err != nil { t.Fatal(err) }

    client := anvil.Client()
    ledger := eth.NewLedger(client)
    // ... run a scenario ...
}
```

Implementation notes:

- Pick a free port (use `net.Listen("tcp", ":0")` and grab the port).
- Spawn `anvil --port <port> --chain-id 31337` as a subprocess.
- Wait for it to be ready: poll `eth_chainId` until it returns.
- `Deploy()` invokes `forge script ../../../contracts/script/DeployAnvil.s.sol --rpc-url <url> --broadcast --private-key <funder>`. Parse the broadcast log for deployed addresses.
- Use `t.Cleanup` to send SIGTERM to anvil, capture stderr if the test fails.

### Funded accounts

Anvil ships with 10 pre-funded accounts (the deterministic dev mnemonic).
Surface them via `AccountSet` so tests have ready-to-use signers without
each test re-deriving keys.

## 2. Sepolia deploy scripts

`core/deploy/v1/sepolia/deploy.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

: "${SEPOLIA_RPC:?set Base Sepolia RPC URL}"
: "${DEPLOYER_KEY:?set deployer private key}"
: "${SAFE_ADDRESS:?set Gnosis Safe address}"

cd "$(dirname "$0")/../../../contracts"

forge script script/DeploySepolia.s.sol \
    --rpc-url "$SEPOLIA_RPC" \
    --private-key "$DEPLOYER_KEY" \
    --broadcast --verify \
    --etherscan-api-key "${BASESCAN_API_KEY:-}" \
    -vvv

# Extract and record deployed addresses
forge script script/DeploySepolia.s.sol \
    --rpc-url "$SEPOLIA_RPC" \
    --sig "exportAddresses()" > /tmp/addrs.json

jq '.deployed' /tmp/addrs.json > ../deploy/v1/sepolia/addresses.json
git add ../deploy/v1/sepolia/addresses.json
echo "Sepolia deploy complete; addresses recorded."
```

`core/deploy/v1/sepolia/bootstrap.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

cd "$(dirname "$0")/../../.."
GOWORK=off go run ./cmd/p2p-bootstrap genesis \
    --config deploy/v1/sepolia/bootstrap.yaml \
    --rpc "$SEPOLIA_RPC" \
    --key-env FOUNDER_PRIVATE_KEY
```

Document the manual steps in `core/deploy/v1/sepolia/README.md`:

1. Get Base Sepolia ETH from a faucet (e.g., `https://www.alchemy.com/faucets/base-sepolia`)
2. Deploy a Gnosis Safe via Safe UI on Base Sepolia, note address
3. Set env vars (`SEPOLIA_RPC`, `DEPLOYER_KEY`, `SAFE_ADDRESS`, `FOUNDER_PRIVATE_KEY`)
4. Run `deploy.sh` then `bootstrap.sh`
5. Verify with `cast call` against the deployed contracts

## 3. Real-engine integration test

`core/p2p/eth/integration_v1_test.go`:

```go
//go:build integration

package eth_test

// Mirrors v0's TestRealEngineIntegration but with on-chain backings.
// Spawns Anvil, deploys contracts, bootstraps genesis, runs 100 jobs
// (90 honest + 10 malicious), asserts:
//   - 90 honest jobs settle on-chain (gross paid - network tax)
//   - 10 malicious jobs full-slashed; reserve grew accordingly
//   - Workers' on-chain reputation reflects deltas
//   - Telemetry events emitted on-chain match Go-side expectations
//   - Credit consumption + repayment cycles correctly for jobs that
//     drew on the founder's per-publisher credit line
```

Pattern:

```go
func TestRealEngineIntegrationV1(t *testing.T) {
    anvil := testharness.StartAnvil(t, testharness.DefaultOpts())
    anvil.Deploy(t)
    anvil.BootstrapGenesis(t, testharness.DefaultGenesis())

    client := anvil.Client()
    ledger := eth.NewLedger(client)
    fed := eth.NewFederation(client)
    auditEng := audit.NewEngine()
    auditEng.SetOnchainPoster(eth.NewAuditPoster(client))

    o := &orchestrator.Orchestrator{
        Ledger:     ledger,
        Audit:      auditEng,
        Federation: fed,
        Provider:   buildRoutingProvider(),  // routing honest vs malicious based on JobID prefix
        Config:     orchestrator.DefaultConfig(),
    }

    // ... pre-fund and bootstrap workers (call p2p-bootstrap to mint
    //     identities via donation jobs, or pre-seed via genesis call) ...

    // Run MixedScenario(100, 10) and assert.
}
```

The scenario adapts the v0 `MixedScenario` factory to:
- Use real Anvil-funded EOAs as workers/publishers
- Use the founder's per-client credit line for the first few jobs (verify credit consume/repay round-trips on-chain)
- Verify on-chain state via direct contract reads (`getStake`, `getReputation`, etc.)

Assertions are the same shape as v0's, plus:
- `InsuranceReserve()` grew by approximately `90 × networkTax + 10 × lockedStake`
- Each malicious worker's `defaultCount` incremented
- The audit dispute window correctly delays finalization (test should fast-forward Anvil's clock via `evm_increaseTime`)

## 4. CI integration

`.github/workflows/v1-tests.yml` (or whatever the project uses):

```yaml
name: v1 tests
on: [push, pull_request]
jobs:
  anvil-integration:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: { submodules: recursive }
      - uses: actions/setup-go@v5
        with: { go-version: '1.25' }
      - name: Install Foundry
        uses: foundry-rs/foundry-toolchain@v1
      - name: Run unit tests
        run: |
          cd core
          GOWORK=off go test ./p2p/eth/... ./p2p/orchestrator/... ./cmd/p2p-bootstrap/...
      - name: Run integration tests (Anvil)
        run: |
          cd core
          GOWORK=off go test -tags integration ./p2p/eth/...
      - name: Run Foundry tests
        run: |
          cd core/contracts
          forge test -vvv
```

Optionally, a separate `nightly-sepolia` workflow that runs against
Sepolia with a long-lived deployer key (stored as GitHub secret). v1.0
can ship without this; it can be added in a follow-up.
