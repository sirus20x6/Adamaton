## Your role

You ship the bootstrap-and-identity-registration glue. Specifically:

1. **New `JobCategory.JobIdentityRegistration`** with a special audit
   path: passing a registration audit triggers `Federation.addDonationCredit`
   and `Registry.mintIdentity` on-chain. Failing it consumes the
   donation stake without minting (the work was wasted, not "free").

2. **Orchestrator lifecycle changes** for credit lines: before locking
   stake, check `EffectiveReputationCap(worker) + ConsumeCredit(publisher, worker, gross)`;
   on settle, `RepayCredit(publisher, worker, gross)`; on full-slash, the
   credit is consumed but never repaid (publisher eats it).

3. **Founder-genesis Go binary** at `core/cmd/p2p-bootstrap/`:
   one-shot CLI that calls `Federation.bootstrapGenesis`, `Registry.mintIdentity`
   (the genesis path), `Federation.setPerClientCredit`, and a stake
   deposit. Configurable via flags or a `bootstrap.yaml` file.

4. **End-to-end tests** for the identity-registration flow: a new node
   submits a registration job, audit replays it, mint happens, the new
   node can then take a paid job.

You are the **only** orchestrator allowed to edit files under
`core/p2p/orchestrator/` (for v1.0's lifecycle changes) and
`core/cmd/p2p-bootstrap/`. You also edit `core/p2p/types/types.go` to
add the new JobCategory.

## Scope (in)

### 1. New `JobCategory` and registration lifecycle

In `core/p2p/types/types.go`, add:

```go
const (
    JobEmbedding            JobCategory = "embedding"
    JobRerank               JobCategory = "rerank"
    JobIdentityRegistration JobCategory = "identity-registration"  // NEW
)
```

The identity-registration job's `Payload` carries a registration blob
(node's intent + signed proof that the submitter controls the address;
shape TBD by you — keep it minimal for v1.0, e.g., `(node_addr,
nonce, sig)`). `Value` is the donation amount in gwei. `Worker` is the
**registering node itself** — they're doing donation work to mint their
own identity.

### 2. Orchestrator changes

In `core/p2p/orchestrator/orchestrator.go`, the `Submit` lifecycle adds
credit-check steps:

```
Step 2.5 (new): Rep-cap check WITH credit
  - repCap := Federation.EffectiveReputationCap(worker)
  - if repCap < jobValueGwei:
      - try Federation.ConsumeCredit(publisher, worker, jobValueGwei - repCap)
      - if consume fails → reject job
      - else: remember (publisher, debt = jobValueGwei - repCap) for repayment on settle

Step 8.5 (new): On clean settle (slashFraction == 0):
  - if debt was consumed earlier:
      - Federation.RepayCredit(publisher, worker, debt)

Step 8.6 (new): On full slash (slashFraction >= 1):
  - debt is NOT repaid (publisher eats the loss up to their credit policy)
  - Emit telemetry event "credit_defaulted"
```

Partial slashes (severity in (0, 1)): proportional credit-repay
(`repaid = debt * (1 - slashFraction)`).

### 3. Identity-registration lifecycle

In the orchestrator, when `job.Category == JobIdentityRegistration`:

- The worker is the registering node itself (not separate worker/publisher).
- Skip the rep-cap check (the node has no identity yet).
- Lock the donation amount from the registering node's deposited stake.
  (They must have deposited ETH for this — handled by the user-facing
  client before calling Submit.)
- Run the audit through the existing path.
- On audit PASS:
  - Settle is special: the donation is NOT paid to the worker. Instead:
    - `Federation.AddDonationCredit(node, donationGwei)`
    - The on-chain finalize triggers `Registry.mintIdentity(node, donationProof)`
    - The donation stake is released back to the node's available
      (they paid in compute, not in dollars; the donation was the work,
      not the money).
- On audit FAIL:
  - Full slash. Donation stake → insurance reserve. Identity NOT minted.

Add an enum or const for the registration-specific severity threshold;
v1.0 default: any severity > 0.5 fails the registration (stricter than
paid-job audits because identity creation is a one-shot trust event).

### 4. Founder-genesis Go binary

`core/cmd/p2p-bootstrap/main.go`:

```go
package main

// Usage:
//   p2p-bootstrap deploy   --config bootstrap.yaml --rpc <url> --key-file <path>
//   p2p-bootstrap genesis  --config bootstrap.yaml --rpc <url> --key-file <path>
//   p2p-bootstrap status   --config bootstrap.yaml --rpc <url>
//
// deploy:  invokes Part A's forge scripts to deploy contracts
// genesis: invokes BootstrapGenesis.s.sol, then post-config (load credit policies, etc.)
// status:  reads on-chain state, prints founder rep, stake, etc.
```

`bootstrap.yaml` example:

```yaml
chain:
  rpc: "http://localhost:8545"          # Anvil; override for Sepolia
  chain_id: 31337                       # Anvil; 84532 for Sepolia

founder:
  address: "0x..."                      # founder EOA
  key_env: "FOUNDER_PRIVATE_KEY"        # or key_file

genesis:
  premine_earned_score: 100
  premine_donated_eth: "1"              # 1 ETH
  working_stake_eth: "5"                # 5 ETH

credit_policy:
  per_node_max_eth: "0.005"             # 0.005 ETH per new node
  total_exposure_eth: "1"               # 1 ETH total

network_params:
  min_identity_creation_donation_eth: "0.01"
  network_tax_bps: 100                  # 1.00%
  verifier_quorum: 2

safe:
  address: "0x..."                      # Gnosis Safe address (deployed separately)
```

Behavior:

- `deploy`: shells out to `forge script script/Deploy.s.sol --rpc-url ... --broadcast`. Captures deployed contract addresses, writes them back to the config.
- `genesis`: shells out to `forge script script/BootstrapGenesis.s.sol ...`. Then makes Go-side calls to wire any off-chain config (none in v1.0).
- `status`: reads founder's `EffectiveReputationCap`, `getStake`, `creditPolicy`, prints a summary.

### 5. End-to-end tests

`core/p2p/orchestrator/registration_test.go` (build tag: `integration`):

- Start a fresh Anvil
- Deploy contracts via `p2p-bootstrap deploy`
- Bootstrap genesis with founder
- A second EOA (the "new node") deposits stake
- Orchestrator submits a `JobIdentityRegistration` job from the new node
- Audit replay passes (using HonestMock from v0)
- Assert: `Registry.isMinted(newNode) == true`; `Federation.GetReputation(newNode).DonatedVerified > 0`
- New node submits a 0.001 ETH paid job, takes the work, settles
- Assert: rep cap grew, stake balance reflects settlement minus network tax

A second test for the failed registration:

- New node submits registration with a `MaliciousMock` provider
- Audit fails (severity ~1.0)
- Assert: identity NOT minted, donation stake routed to reserve, node still has no identity

Use Part D's Anvil helpers if available (`testharness/anvil.go`); if Part
D hasn't committed yet, roll your own minimal Anvil bringup for this test
and replace it later.
